# AGENTS.md ‚Äî English Work Tracker

Offline-first PWA for professionals who want to track consistent English learning.
Built with TanStack Router/Start, React 19, Dexie (IndexedDB), Zustand, Tailwind CSS 4, and Vite.

---

npm run lint       # ESLint + auto-fix (.ts, .tsx)

## Database

**Engine:** Dexie 4 over IndexedDB. DB name: `EnglishWorkTracker`.

**Tables:**

| Table | Key fields | Notes |
|---|---|---|
| `plans` | `id`, `name`, `daily_goal_minutes` | Weekly template stored as JSON |
| `daily_blocks` | `[plan_id+date]`, `activity_type` | 30+ min/day = valid streak day |
| `writing_entries` | `[plan_id+date]`, `word_count` | Auto-saved text + active time |
| `resource_categories` | `id`, `plan_id`, `name` | User-defined categories |
| `resources` | `id`, `plan_id`, `category_id`, `url` | Study links |
| `monthly_reviews` | `[plan_id+year+month]` | Self-evaluations |

Schema lives in [src/db/index.ts](src/db/index.ts). Migrations are version-gated ‚Äî add a new `db.version(N)` block, never change existing ones.

### Rules for DB access

- **Writes** must go through a service in `src/services/`.
- **Reads** should use the hooks in `src/db/hooks.ts` (live, reactive) or a service method for one-off reads.
- Use Dexie transactions (`db.transaction`) for multi-table atomic writes.
- All timestamps are Unix milliseconds (`Date.now()`).

## Routing

TanStack Router with file-based routing. `routeTree.gen.ts` is auto-generated ‚Äî run `npm run dev` or `npm run build` to regenerate it after adding/removing route files. Never edit it manually.

Add a new page by creating `src/routes/yourpage.tsx` and exporting a `Route` using `createFileRoute`.

## Styling

- **Tailwind CSS 4** ‚Äî utility-first; no custom CSS classes in components.
- **No hardcoded colors.** Every color in JSX/TSX must use a Tailwind semantic utility (`text-on-surface`, `bg-primary-dark`, `border-outline-variant`, etc.) backed by a CSS variable in `src/styles.css`. Raw hex values (`#xxx`), `rgb()`, `rgba()`, and inline `style` color props are forbidden. SVG `stroke`/`fill` attributes must use `currentColor` or `var(--color-*)` ‚Äî never a raw hex. Only exception: browser-level HTML meta tags (e.g. `theme-color`) which require a concrete color string. To add a new color: define a CSS variable in `:root`, register it in the `@theme inline` block ‚Äî never use it raw anywhere.
- Responsive breakpoint: `md:` (768px) ‚Äî sidebar on desktop, bottom nav on mobile.
- Global styles, design tokens (`:root`), and Tailwind `@theme inline` mapping in [src/styles.css](src/styles.css).

## State Management

Single Zustand store at [src/store/profile.ts](src/store/profile.ts):
- Holds `UserProfile` and the active `StudyPlan`.
- Provides async action methods that persist to IndexedDB.
- Initialized once via `_init()` in `src/routes/__root.tsx` on app mount.

Use `useProfileStore()` in components. Do not add separate stores for other data ‚Äî prefer `useLiveQuery` hooks.

## Common Tasks

**Add a new activity type**
1. Update `ActivityType` union in [src/services/types.ts](src/services/types.ts).
2. Update the chip color map in [src/components/ui/ActivityTypeChip.tsx](src/components/ui/ActivityTypeChip.tsx).

**Add a new DB table**
1. Define the interface in [src/db/index.ts](src/db/index.ts).
2. Add to the Dexie schema inside a new `db.version(N).stores({...})` block.
3. Create a service file in `src/services/`.
4. Add live-query hooks to [src/db/hooks.ts](src/db/hooks.ts) as needed.

**Add a new route/page**
1. Create `src/routes/yourpage.tsx` with `createFileRoute('/yourpage')`.
2. Restart dev server to regenerate `routeTree.gen.ts`.
3. Add navigation entry in [src/components/Sidebar.tsx](src/components/Sidebar.tsx) and [src/components/BottomNav.tsx](src/components/BottomNav.tsx).

**Streak logic**
- A day is valid when total block duration ‚â• 30 minutes.
- Today may be incomplete and should not break the streak.
- See [src/lib/streak.ts](src/lib/streak.ts) for all date and streak utilities.
